node2vec : Scalable Feature Learning for Networks

1. Introduction：

  1. 介绍网络分析的任务

    • 网络节点的分类(node classification task):将网络中的节点进行聚类，哪些节点有着相同的属性，就分为一类。
    • 链接预测(Link prediction)：预测网络中哪些顶点有着潜在的关联。
    • …

  2. 网络嵌入问题

    完成上述任务的前提是解决网络嵌入问题，以前的手工提取特征的方式有着诸多不足。

    • 解优化函数的方式学习网络的表示特征（learn feature representations by solving an optimization problem)：面临计算效率和准确度的平衡问题，无法兼顾两者。使用传统的降维方法如PCA（主成分分析）也确实有一定效果，但是会涉及矩阵分解，运算量大准确率也不高。
    • 本文尝试设计出既能保持节点邻居特性而且又特别容易训练的模型。我们发现很多节点在网络中往往有一些类似的结构特征。一种结构特征是很多节点会聚集在一起，内部的连接远比外部的连接多，我们称之为社区（community）；另一种结构特征是网络中两个可能相聚很远的点在边的连接上有着相似的特征。
    • 如下图，u,s1,s2,s4,s4u,s_1,s_2,s_4,s_4u,s1​,s2​,s4​,s4​属于一个社区，u,s6u,s_6u,s6​在结构上有着相似的特征。

要设计的网络表示学习算法的目标必须满足这两点：

  • 同一个社区内节点表示相似。
  • 拥有类似结构特征的节点表示相似。

总之本文的贡献是：提出了一个有效的，可扩展的表示学习算法，可以体现网络特征和节点邻居特征。

2. Related Works :

之前的工作存在以下问题：

  • 特征需要依赖人手工定义，不是某个领域内的专业人士无法从事该工作。
  • 一些无监督学习（Unsupervised feature learning）中的降维方法拿来使用，但是这些方法计算效率低，准确度也不够，而且还不能反映出网络的结构特征。

自然语言处理（natural language processing）表征学习的进展为本文的作者提供了思路。与deepwalk相似，这里也用skip-gram模型来解决网络表示学习的问题。

3.0 Negative sampling

负采样（negative sampling）是用来提高训练速度并且改善所得到词向量的质量的一种方法。不同于原本每个训练样本更新所有的权重，负采样每次让一个训练样本仅仅更新一小部分的权重，这样就会降低梯度下降过程中的计算量。

当我们用训练样本 ( input word: “fox”，output word: “quick”) 来训练我们的神经网络时，“ fox”和“quick”都是经过one-hot编码的。如果我们的vocabulary大小为10000时，在输出层，我们期望对应“quick”单词的那个神经元结点输出1，其余9999个都应该输出0。在这里，这9999个我们期望输出为0的神经元结点所对应的单词我们称为“negative” word。

当使用负采样时，我们将随机选择一小部分的negative words（比如选5个negative words）来更新对应的权重。我们也会对我们的“positive” word进行权重更新（在我们上面的例子中，这个单词指的是”quick“）。

3.1 Classic search strategies

  • BFS & DFS
  • BFS倾向于在初始节点的周围游走，可以反映出一个节点的邻居的微观特性；而DFS一般会跑的离初始节点越来越远，可以反映出一个节点邻居的宏观特性。

如上图，对于一个随机游走，如果已经采样了(t,v)(t,v)(t,v),也就是说现在停留在节点v上，那么下一个要采样的节点x是哪个？作者在这里定义了一个概率分布，也就是一个节点到它的不同邻居的转移概率

  • 参数解释:
    • 返回概率p:
      • 如果p>max(q,1)p>max(q,1)p>max(q,1),那么采样会尽量不往回走,对应上图的情况,就是下一个节点不太可能是上一个访问的节点t
      • 如果p<min(q,1)p<min(q,1)p<min(q,1),那么采样会更倾向于返回上一个节点,这样就会一直在起始点走位某些节点来回转来转去.
    • 出入概率q:
      • 如果q>1q>1q>1,那么游走会倾向于在起始点周围的节点跑,反映一个节点的BFS特性.
      • 如果q<1q<1q<1,那么游走会倾向于往远处跑,反映出DFS特性.
  • 分布解释:
    • 如果t与x相等,那么采样x的概率为1p\frac{1}{p}p1​;
    • 如果t与x相连,那么采样x的概率为111;
    • 如果t与x不相连,那么采样x的概率为1q\frac{1}{q}q1​;

特殊地,当p=1,q=1时,游走方式就等同于DeepWalk中的随机游走.

4. Algorithm

  • LearnFeatures

    1. 根据p,q和之前的概率公式计算一个节点到它的邻居的转移概率

    2. 将这个转移概率加入图G形成G’

    3. walks储存随机游走,先初始化为空

    4. 外循环r次表示每个节点作为初始节点要生成r个随机游走

    5. 对图中的每一个节点
      6. 生成一个随机游走walk
      7. 将walk添加入walks

    6. 用SGD的方法对walks进行训练

  • Node2vecWalk

上述第六步中一条walk形成方式:

  1. 将初始节点u添加入walk
  2. 循环添加入l−1l-1l−1个节点
    3. 当前节点设为walk最后添加的节点.
    4. 找出当前节点的所有邻居节点.
    5. 根据转移概率采样选择某个邻居s.
    6. 将该邻居加入walk中.

5. Conclusion

本文主要思想与deepwalk一致,不过改进了随机游走的生成方式,充分反映了DFS & BFS 两种采样的特性,从而提高网络嵌入的效果。
